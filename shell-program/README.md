# SWU Shell

C언어로 구현한 간이 Unix 셸 프로그램입니다. `fork`/`execvp` 기반의 프로세스 생성과 파일 디스크립터 조작을 통해 명령어 실행, 입출력 리다이렉션, 파이프를 지원합니다.

## 빌드 및 실행

```bash
gcc -o swu_shell swu_2023111394.c
./swu_shell
```

실행하면 `SWU Shell! ` 프롬프트가 출력되며, 명령어를 입력할 수 있습니다.

## 구현된 기능

| 기능 | 문법 | 설명 |
|------|------|------|
| 명령어 실행 | `ls`, `cat file` 등 | `fork` + `execvp`로 외부 명령어 실행 |
| 입력 리다이렉션 | `cmd < file` | 파일을 표준 입력으로 연결 |
| 출력 리다이렉션 (덮어쓰기) | `cmd > file` | 표준 출력을 파일로 연결 (O_TRUNC) |
| 출력 리다이렉션 (추가) | `cmd >> file` | 표준 출력을 파일 끝에 추가 (O_APPEND) |
| 입출력 동시 리다이렉션 | `cmd < in > out` | 입력과 출력을 동시에 파일로 연결 |
| 파이프 | `cmd1 \| cmd2` | 첫 번째 명령의 출력을 두 번째 명령의 입력으로 연결 |
| 파이프 + 리다이렉션 | `cmd1 \| cmd2 > file` | 파이프와 리다이렉션 조합 |
| 종료 | `exit` | 셸 종료 |

## 핵심 함수

- `shell_loop()` — 메인 루프 (프롬프트 출력 -> 입력 읽기 -> 파싱 -> 실행)
- `read_input()` — `getline()`으로 사용자 입력 읽기
- `parse_input()` — `strtok()`으로 공백 기준 토큰 분리
- `execute_command()` — `<`, `>`, `>>`, `|` 기호를 감지하여 적절한 핸들러 호출
- `handle_redirectIn()` — 입력 리다이렉션 처리 (`dup2`로 stdin 교체)
- `handle_redirectOut()` — 출력 리다이렉션 처리 (`dup2`로 stdout 교체)
- `handle_redirectInandOut()` — 입출력 동시 리다이렉션
- `handle_pipes()` — `pipe()` + `fork()` 두 번으로 두 명령어 연결

## 제약 사항

- 최대 입력 길이: 1024자
- 최대 인자 수: 64개 (동적 확장 가능)
- 파이프는 2개 명령어 간 단일 파이프만 지원
